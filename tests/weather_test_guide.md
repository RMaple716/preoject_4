# 天气API集成测试指南

## 测试文件说明

### 1. test_weather_integration.py
主要的测试脚本，包含以下测试内容：
- 天气服务功能测试
- 景点智能体集成天气API测试
- 行程中天气信息集成测试

### 2. run_weather_test.bat
Windows批处理文件，用于快速运行测试脚本

## 测试前准备

### 1. 配置高德地图API密钥
在运行测试前，需要配置高德地图API密钥：

**Windows:**
```bash
set AMAP_API_KEY=your_api_key_here
```

**Linux/Mac:**
```bash
export AMAP_API_KEY=your_api_key_here
```

### 2. 安装依赖
确保已安装所有必要的Python依赖：
```bash
pip install -r requirements.txt
```

## 运行测试

### 方法1: 使用批处理文件（Windows）
双击运行 `run_weather_test.bat` 或在命令行中执行：
```bash
tests\run_weather_test.bat
```

### 方法2: 直接运行Python脚本
```bash
python tests\test_weather_integration.py
```

## 测试内容说明

### 测试1: 天气服务功能
- 测试获取指定城市的天气预报
- 验证返回数据的格式和内容
- 显示未来几天的天气情况

### 测试2: 景点智能体集成
- 测试景点推荐时是否考虑天气因素
- 验证天气信息是否正确传递给LLM
- 检查推荐结果的合理性

### 测试3: 行程天气信息
- 验证天气信息在行程中的存储格式
- 检查天气数据的完整性

## 预期结果

### 成功情况
- 所有测试显示"✓ 通过"
- 能够获取到天气预报数据
- 景点推荐考虑了天气因素
- 行程中包含完整的天气信息

### 失败情况
- 未配置API密钥：部分测试会失败
- 网络问题：无法获取天气数据
- 城市名称错误：API无法识别

## 常见问题

### 1. 提示"未配置AMAP_API_KEY环境变量"
**解决方法：** 设置环境变量，参考"测试前准备"部分

### 2. 提示"获取天气信息失败"
**可能原因：**
- API密钥无效
- 网络连接问题
- 城市名称不被识别

**解决方法：**
- 检查API密钥是否正确
- 检查网络连接
- 尝试使用其他城市名称（如：北京、上海）

### 3. 测试脚本运行失败
**可能原因：**
- Python版本过低（需要3.8+）
- 缺少必要的依赖包

**解决方法：**
- 升级Python版本
- 安装所有依赖包

## 注意事项

1. 测试脚本会调用真实的高德地图API，请确保：
   - API密钥有效
   - 有足够的API调用额度
   - 网络连接正常

2. 测试过程中可能会产生API调用费用，请注意监控使用量

3. 建议在非生产环境下运行测试

## 扩展测试

如果需要测试更多城市或场景，可以修改 `test_weather_integration.py` 中的测试参数：
- 修改 `city_name` 测试不同城市
- 修改 `travel_days` 测试不同天数的行程
- 修改 `travel_date` 测试不同日期的天气
